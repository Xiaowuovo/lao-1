## 📱 功能说明

### 1. 用户注册

**访问**: `http://localhost:8080/register.html`

**注册信息**:
- ✅ **学号**（必填）- 12位数字，系统自动验证格式和学院代码
- ✅ **用户名**（必填）- 2-20字符，可中英文
- ✅ **真实姓名**（必填）- 用于身份确认
- ✅ **邮箱**（必填）- 接收系统通知
- ⭕ **手机号**（可选）
- ✅ **密码**（必填）- 至少6位，支持强度检测

**注册流程**:
1. 填写注册信息
2. 系统自动验证学号格式
3. 根据学号自动识别学院和年级
4. 创建用户账号和默认设置
5. 跳转到登录页面

### 2. 用户登录

**访问**: `http://localhost:8080/login.html`

**登录方式**: 学号 + 密码

**功能**:
- ✅ 记住我（保存学号）
- ✅ 会话管理（24小时有效期）
- ✅ 自动跳转到首页
- ✅ 会话过期自动提醒

### 3. 用户中心

**个人资料**:
- 查看学号、姓名、学院信息
- 修改邮箱、手机号
- 查看登录次数和最后登录时间

**修改密码**:
- 需要输入原密码
- 新密码至少6位
- 修改后需重新登录

### 4. 数据隔离

**每个用户拥有独立的**:
- ✅ 事件记录
- ✅ 提醒任务
- ✅ 课表安排
- ✅ 历史归档
- ✅ 用户设置
- ✅ 上传文件记录

**用户之间数据完全隔离，互不干扰**

## 🔒 安全机制

### 1. 密码安全
- 密码使用MD5加密存储（示例，实际建议用bcrypt）
- 密码长度至少6位
- 支持密码强度检测

### 2. 会话管理
- 使用session_token进行身份验证
- 会话有效期24小时
- 支持主动登出
- 过期自动失效

### 3. 权限控制
- 所有API接口需要登录认证
- 用户只能访问自己的数据
- 数据库级别外键约束

### 4. 操作日志
- 记录用户登录/登出
- 记录注册时间
- 记录重要操作

## 🛠️ 技术实现

### 后端架构

```
backend/
├── auth.py                 # 认证管理器
│   ├── AuthManager         # 认证核心类
│   ├── register()          # 用户注册
│   ├── login()             # 用户登录
│   ├── verify_session()    # 会话验证
│   ├── logout()            # 用户登出
│   └── change_password()   # 修改密码
│
└── api_with_auth.py        # 带认证的API
    ├── @login_required     # 登录装饰器
    ├── /api/auth/register  # 注册接口
    ├── /api/auth/login     # 登录接口
    ├── /api/auth/logout    # 登出接口
    └── /api/auth/profile   # 用户资料接口
```

### 前端架构

```
frontend/
├── login.html/js           # 登录页面
├── register.html/js        # 注册页面
├── auth-check.js           # 认证检查模块
│   ├── checkAuth()         # 检查登录状态
│   ├── getCurrentUser()    # 获取当前用户
│   ├── logout()            # 登出
│   └── authFetch()         # 带认证的请求
│
└── profile.html            # 用户中心（待创建）
```

### 数据库设计

**新增表**:
1. **user_sessions** - 用户会话表
2. **user_activity_logs** - 操作日志表
3. **college_codes** - 学院代码表

**扩展字段**:
- users表新增：student_id, real_name, college_code, class_code, grade等

**存储过程**:
- `sp_record_login()` - 记录登录
- `sp_user_register()` - 用户注册

**函数**:
- `fn_validate_student_id()` - 验证学号格式

## 📊 API接口文档

### 认证相关

#### 1. 用户注册
```http
POST /api/auth/register
Content-Type: application/json

{
  "student_id": "202205570603",
  "username": "zhangsan",
  "real_name": "张三",
  "password": "xtu123456",
  "email": "zhangsan@xtu.edu.cn",
  "phone": "13800000001"
}
```

#### 2. 用户登录
```http
POST /api/auth/login
Content-Type: application/json

{
  "student_id": "202205570603",
  "password": "xtu123456"
}
```

**响应**:
```json
{
  "success": true,
  "message": "登录成功",
  "session_token": "xxx",
  "user_info": {
    "user_id": 2,
    "student_id": "202205570603",
    "username": "zhangsan",
    "real_name": "张三",
    "college_name": "信息工程学院",
    "role": "user"
  }
}
```

#### 3. 用户登出
```http
POST /api/auth/logout
Authorization: Bearer {session_token}
```

#### 4. 获取用户资料
```http
GET /api/auth/profile
Authorization: Bearer {session_token}
```

#### 5. 修改密码
```http
POST /api/auth/changePassword
Authorization: Bearer {session_token}
Content-Type: application/json

{
  "old_password": "xtu123456",
  "new_password": "newpassword"
}
```

### 业务接口（需要登录）

所有业务接口都需要在请求头中添加：
```http
Authorization: Bearer {session_token}
```

系统会自动获取当前登录用户ID，确保数据隔离。

## 🔧 常见问题

### Q1: 忘记密码怎么办？
**A**: 联系管理员重置密码，或直接在数据库中修改：
```sql
UPDATE users 
SET password_hash = MD5('新密码') 
WHERE student_id = '学号';
```

### Q2: 学号格式验证失败？
**A**: 确保：
- 学号为12位数字
- 年份在2015-2030之间
- 学院代码在college_codes表中存在

### Q3: 会话过期后怎么办？
**A**: 系统会自动跳转到登录页，重新登录即可。

### Q4: 如何添加新学院代码？
**A**: 在college_codes表中插入：
```sql
INSERT INTO college_codes (college_code, college_name, college_short_name)
VALUES ('0599', '新学院', '新院');
```

## 📝 开发者注意事项

### 前端开发
1. 所有页面（除登录注册）都需引入 `auth-check.js`
2. 使用 `authFetch()` 发起API请求
3. 使用 `getCurrentUser()` 获取当前用户信息

### 后端开发
1. 新接口使用 `@login_required` 装饰器
2. 从 `request.current_user` 获取当前用户信息
3. 所有数据库操作都要带上 `user_id` 条件

### 数据库维护
1. 定期清理过期会话
2. 定期备份用户数据
3. 监控操作日志
