# 邮件提醒功能使用说明

## 功能说明

系统支持通过邮件发送事件提醒。当提醒时间到达时，系统会自动向用户的注册邮箱发送提醒邮件。

## 配置步骤

### 1. 配置邮件服务器

复制 `.env.example` 为 `.env`，并填写邮件配置：

```bash
cp .env.example .env
```

编辑 `.env` 文件，配置邮件服务器信息：

#### QQ邮箱配置示例：
```
SMTP_SERVER=smtp.qq.com
SMTP_PORT=587
SMTP_USERNAME=your_qq_email@qq.com
SMTP_PASSWORD=your_qq_auth_code
SMTP_FROM_EMAIL=your_qq_email@qq.com
```

#### 163邮箱配置示例：
```
SMTP_SERVER=smtp.163.com
SMTP_PORT=465
SMTP_USERNAME=your_163_email@163.com
SMTP_PASSWORD=your_163_auth_code
SMTP_FROM_EMAIL=your_163_email@163.com
```

**重要提示：**
- `SMTP_PASSWORD` 应填写邮箱的**授权码**，而非登录密码
- QQ邮箱授权码获取：邮箱设置 → 账户 → POP3/IMAP/SMTP服务 → 生成授权码
- 163邮箱授权码获取：邮箱设置 → POP3/SMTP/IMAP → 开启服务并获取授权密码

### 2. 用户设置

用户需要在系统中：
1. 确保个人资料中填写了有效的邮箱地址
2. 在"设置与统计"页面开启"邮件提醒"功能

### 3. 启动定时任务

邮件提醒需要后台定时任务支持。有两种方式：

#### 方式一：手动启动定时任务（推荐用于开发测试）

在backend目录下运行：
```bash
python reminder_scheduler.py
```

该脚本会每分钟检查一次待发送的提醒邮件。

#### 方式二：使用系统定时任务（推荐用于生产环境）

**Windows系统 - 使用任务计划程序：**
1. 打开"任务计划程序"
2. 创建基本任务
3. 触发器：每1分钟重复一次
4. 操作：启动程序 `python`
5. 参数：`reminder_scheduler.py`
6. 起始于：backend目录路径

**Linux系统 - 使用crontab：**
```bash
# 编辑crontab
crontab -e

# 添加以下行（每分钟执行一次）
* * * * * cd /path/to/backend && python reminder_scheduler.py >> /var/log/reminder.log 2>&1
```

## 测试邮件功能

系统提供了测试邮件API，可以在前端调用测试：

```javascript
fetch('http://localhost:5000/api/testEmail', {
    method: 'POST',
    headers: getAuthHeaders()
})
```

或在设置页面添加"测试邮件"按钮。

## 工作流程

1. 用户创建事件并设置提醒
2. 系统在 `reminder_tasks` 表中创建提醒记录，状态为 `pending`
3. 定时任务每分钟检查一次，查找 `reminder_time <= NOW()` 且状态为 `pending` 的提醒
4. 对于开启了邮件提醒的用户，发送邮件
5. 发送成功后，将提醒状态更新为 `sent`

## 故障排查

### 邮件发送失败

1. **检查邮件配置**
   - 确认 `.env` 文件中的配置正确
   - 确认使用的是授权码而非登录密码

2. **检查网络连接**
   - 确保服务器可以访问SMTP服务器
   - 某些网络可能屏蔽SMTP端口

3. **查看日志**
   - 检查 `reminder_scheduler.py` 的输出日志
   - 检查Flask应用的日志

### 用户收不到邮件

1. **检查用户设置**
   - 确认用户邮箱地址正确
   - 确认用户开启了邮件提醒功能

2. **检查垃圾邮件**
   - 提醒用户检查垃圾邮件文件夹
   - 建议用户将发件地址加入白名单

3. **检查提醒时间**
   - 确认提醒时间已经到达
   - 检查 `reminder_tasks` 表中的记录状态

## API接口

### 发送待发送的提醒（后台任务调用）
```
POST /api/sendPendingReminders
```

### 测试邮件发送（需要登录）
```
POST /api/testEmail
```

### 健康检查（包含邮件配置状态）
```
GET /api/health
```

返回示例：
```json
{
    "success": true,
    "message": "服务正常",
    "version": "2.0-xtu-enhanced",
    "email_configured": true
}
```
